# 🛡️ লিনাক্স সুডো (Sudo) অ্যাক্সেস ম্যানেজমেন্ট গাইড

প্রোডাকশন এনভায়রনমেন্টে নিরাপত্তার খাতিরে সবাইকে **Root** পাসওয়ার্ড দেওয়া অত্যন্ত ঝুঁকিপূর্ণ। এর পরিবর্তে ইউজারদের তাদের দায়িত্ব অনুযায়ী নির্দিষ্ট ক্ষমতা বা **Sudo Access** দেওয়া হয়। এই গাইডে আমরা শিখব কীভাবে বিভিন্ন লেভেলের অ্যাডমিনিস্ট্রেটিভ প্রিভিলেজ সেটআপ করতে হয়।

---

## 🔑 সুডো (Sudo) কী এবং কেন প্রয়োজন?

* **Sudo (SuperUser Do):** এটি এমন একটি কমান্ড যা সাধারণ ইউজারকে সাময়িকভাবে রুট (Root) ইউজারের ক্ষমতা দিয়ে কোনো কাজ করার অনুমতি দেয়।
* **নিরাপত্তা:** এর মাধ্যমে রুট পাসওয়ার্ড শেয়ার না করেই ইউজারকে কাজ করার সুযোগ দেওয়া যায়। ইউজার নিজের পাসওয়ার্ড ব্যবহার করেই কমান্ড চালাতে পারে।
* **লগিং (Logging):** সুডো ব্যবহার করে কে, কখন, কোন কমান্ড চালিয়েছে তার বিস্তারিত রেকর্ড সিস্টেমে জমা থাকে, যা জবাবদিহিতা নিশ্চিত করে।

---

## 🏗️ ল্যাব সিনারিও এবং সেটআপ

আমরা এই ল্যাবে দুই ধরনের ইউজার এবং তাদের জন্য আলাদা লেভেলের পারমিশন তৈরি করব:
1.  **Super Admin (Full Access):** যারা সিস্টেমের সব কাজ করতে পারবে।
2.  **Service Operator (Limited Access):** যারা শুধু নির্দিষ্ট কিছু সার্ভিস (যেমন: SSH) নিয়ন্ত্রণ করতে পারবে।

### ধাপ ১: ইউজার ও গ্রুপ তৈরি
```bash
# ১. সুপার অ্যাডমিনদের জন্য গ্রুপ তৈরি
sudo groupadd superadmin

# ২. দুইজন আলাদা ইউজার তৈরি
sudo useradd -m serviceop         # সাধারণ ইউজার
sudo useradd -G superadmin -m rootuser  # সুপার অ্যাডমিন গ্রুপের মেম্বার

# ৩. পাসওয়ার্ড সেট করা
sudo passwd serviceop
sudo passwd rootuser
```

## 🛠️ সুডো পারমিশন কনফিগার করা

সুডো পারমিশন মূলত দুটি জায়গায় কনফিগার করা যায়:
* `/etc/sudoers` ফাইলে।
* `/etc/sudoers.d/` ডিরেক্টরির ভেতরে আলাদা ফাইলে (এটি বেশি নিরাপদ ও গোছানো পদ্ধতি)।


### ধাপ ২: ফুল অ্যাডমিনিস্ট্রেটিভ এক্সেস (Superadmin)
আমরা চাই `superadmin` গ্রুপের সবাই যেন সব কমান্ড চালাতে পারে। এর জন্য `visudo` কমান্ড দিয়ে নিচের লাইনটি যোগ করতে হবে:
`%superadmin ALL=(ALL) ALL`

### ধাপ ৩: লিমিটেড এক্সেস (Service Operator)
আমরা চাই `serviceop` ইউজার শুধু SSH সার্ভিস রিস্টার্ট এবং রিলোড করতে পারুক। এর জন্য আমরা একটি কাস্টম ফাইল তৈরি করব:
`sudo visudo -f /etc/sudoers.d/serviceop`

ভেতরে নিচের কোডটি লিখুন:
```bash
# নির্দিষ্ট কমান্ডের জন্য একটি ডাকনাম বা Alias তৈরি
Cmnd_Alias SERVICE = /bin/systemctl restart sshd.service, /bin/systemctl reload sshd.service

# ইউজারকে শুধু ওই Alias টি ব্যবহারের অনুমতি দেওয়া
serviceop ALL=SERVICE
```

## 🧪 এক্সেস যাচাই ও পরীক্ষা (Testing)

### ১. ফুল এক্সেস পরীক্ষা (rootuser)
`rootuser` হিসেবে লগইন করে `sudo cat /etc/shadow` কমান্ডটি দিন। এটি সফল হবে কারণ তার কাছে সব সিস্টেম রিসোর্স দেখার ক্ষমতা আছে।

### ২. লিমিটেড এক্সেস পরীক্ষা (serviceop)
* **সফল কমান্ড:** `sudo systemctl restart sshd.service` (এটি কাজ করবে)।
* **ব্যর্থ কমান্ড:** `sudo systemctl status sshd.service` (এটি ব্যর্থ হবে বা Permission Denied দেখাবে, কারণ তাকে শুধু রিস্টার্ট ও রিলোড করার অনুমতি দেওয়া হয়েছে)।

### ৩. সার্ভিস রিস্টার্ট হয়েছে কি না যাচাই
সার্ভিস রিস্টার্ট হলে তার প্রসেস আইডি (PID) বদলে যায়।
* প্রথমে দেখুন: `pidof sshd` (ধরুন আইডি ৩৩৬৫)
* রিস্টার্ট দিন: `sudo systemctl restart sshd.service`
* আবার দেখুন: `pidof sshd` (আইডি বদলে নতুন কিছু হবে, যেমন ৩৬৮২)

## 💡 সুডো কীভাবে কাজ করে? (কাজের ধাপ)
১. ইউজার কমান্ডের আগে `sudo` লিখলে সিস্টেম প্রথমে কনফিগারেশন ফাইলগুলো চেক করে।
২. ইউজারের ওই কমান্ড চালানোর অনুমতি আছে কি না তা যাচাই করে।
৩. অনুমতি থাকলে ইউজারের কাছে তার **নিজের পাসওয়ার্ড** চায়।
৪. পাসওয়ার্ড সঠিক হলে সাময়িকভাবে রুট পাওয়ার দিয়ে কমান্ডটি এক্সিকিউট করে এবং সাথে সাথে লগ ফাইলে তা লিখে রাখে।
